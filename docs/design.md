## コンセプト

「物理の碁石では表現できなかった 0.5 石を、液体で実現した囲碁」。

サッカーを見て「あとから動かせる囲碁」と感じた観察が起点。**スペースやエリアへの「効き」を可視化したい**、という動機から発展した。

## 世界観 — 石は柵、水は取得済みの地

囲碁の「地（目）」を、**柵で囲って溜めた水**として物理的に表す。碁は「囲い切って初めて地になる」——それを水の物理でそのまま可視化する。

| 要素 | 物質 | 役割 |
| ---- | ---- | ---- |
| 1 石 | 石製の**柵（壁）**。完全に固まった構造物 | 盤上に立つ仕切り。水をせき止める |
| 0.5 石 | まだ固まっていないセメント。柔らかく不安定 | 壊れやすい柵。半透明で揺らぐ |
| 内側の液体（白・黒の流体） | 各色の**取得済みの地（目）** | 柵で囲われた領域に溜まる。**水量(m³) = 地 = スコア** |

**核心: 水は流れ出る。** 一色の柵で**囲い切った**領域だけ水が溜まって残る（＝確保した地）。柵に隙間が1つでもあれば、そこから水が流れ出て下がる（＝まだ地じゃない）。だから状態を**標高（水面の高さ）と流れ**で表す。

- 空盤＝柵なし＝どこも囲われていない＝水は全部流れ出て**乾く**（水はどこにも無い）。
- 石（柵）は水に沈まず盤上に立つ。水は柵の**間（囲まれた領域）**に溜まる。**柵の上に水が乗ることはない**。

## 表示仕様（ビジュアル設計）

**丸い碁石は捨てる。** リキッ碁の見た目は「柵（石）」と「水（地）」の2要素だけ。

### 石＝柵（境界線）

- 石は丸い碁石でなく、**盤に立つ柵（壁・線）**として描く。固体・不透明。
- **隣り合う同色石は実線**でつながる。**斜めの同色石は隙間のある斜め線**（斜めは繋がりきらない＝碁の連結を線の見た目で表す）。
- **1石＝実線の硬い柵**。**0.5石＝半透明で揺らぐ壊れやすい柵**（セメントが固まっていない）。
- **0.5石が消える瞬間＝柵が崩れて水が流れ出す演出**（リーク）。

### 水＝地（標高と流れで表す）

| 囲碁の状態 | 水の表現 | 標高 |
| ---- | ---- | ---- |
| 確定した地（一色で囲い切った） | 海抜0の海・池。液体が完全に溜まった安定状態 | **低い（海抜0）** |
| 勢力が強い場所 | 表面張力で盛り上がった液体 | 高い |
| 境界・接触 | 水際・波打ち際 | ― |
| 漏れそうな薄い囲み（不安定な地） | 高台の水たまり・今にも流れ出しそう。粘性が弱い | **高い** |
| 相手に侵食されている | 粘性が下がって流れ始める | 下がる／流れる |
| 死に石 | 水没 | ― |
| 囲われていない空間 | 水が流れ出て残らない＝乾く | 水なし |

- **高さの意味＝囲いの安定度と流れ**。囲い切った確定地は海抜0の凪の池、薄い囲み・不安定は高く盛り上がって今にも流れ出す、囲いの無い所は乾く。「効きが強い＝盛り上がる」も同じ軸（未確定の勢力は高く、確定すると海抜0へ落ち着く）。
- 水の色＝所有（白っぽい流体 vs 黒っぽい流体）。
- **水量（体積 m³）＝取得済みの地＝スコア**（下記「デジタルならではの解決」）。

## 石の計算ルール — すべて足し算

黒をプラス、白をマイナスとして加算するだけ。セル値は `{-1, -0.5, 0, 0.5, 1}` の5値。

**石が触れ合う経路は2つだけ。** 着手は空き点にしか置けない。既にある石へ直接「置きにいく」ことは無い。

- **① 同時着手** — 両者が同じ空き点を同じターンに奪い合う（ルール①）。デルタが足し合わさる。
- **② ムーブ** — 0.5 が隣へ乗り込む（ルール③）。**1石は動けない**ので、1石のほうから相手の石へ触れにいくことはできない。

この2経路で起こる加算の全パターン:

| ぶつかり方            | 計算            | 結果              | 経路                  | 現象                    |
| --------------------- | --------------- | ----------------- | --------------------- | ----------------------- |
| 空きに1石             | `0 + 1 = 1`     | 1石               | 着手                  | place                   |
| 空きにポア            | `0 + 0.5 = 0.5` | 0.5石             | 着手                  | ポア (pour)             |
| 自分の0.5 + 自分の0.5 | `0.5 + 0.5 = 1` | 1石に確定         | ムーブ                | ソリディファイ          |
| 0.5 + 相手の0.5       | `0.5 - 0.5 = 0` | 両方消滅          | 同時着手 / ムーブ     | **キャンセル (cancel)** |
| 1石 + 相手の1石       | `1 - 1 = 0`     | 両方消滅          | 同時着手              | capture                 |
| 自分の1石 + 相手の0.5 | `1 - 0.5 = 0.5` | 自分の0.5へ削れる | 同時着手 / ムーブ迎撃 | reduce                  |

**1石が0.5とぶつかると必ず0.5に削れる**（1のまま残らない）。相手の0.5の上に1石を置いて色を奪う手は、1が動けず同時着手も空き点限定なので**起こらない**（旧「オーバーポア」は廃止。0.5 同士の相殺は **Cancel** に統一し、旧「ブレイク」も廃止）。

## 3つのルールセット（絡み合って成立する）

液体表示・境界線・3D 地形は**表示の改善**であってルール変更ではない。

### ルール① 同時プロット制

先手・後手なし。両者が次の手を同時に公開する。コミ不要（先手有利が構造的に存在しない）。衝突は全て足し算で処理。

### ルール② 1.5手（星パックマン）

星・天元に置いた瞬間、1.5 手の権利を獲得（ストック、上限 1.5）。星の石が取られても権利は消えない。相手が同じ星に置けば**相手も**（各自の権利を）1.5 に据える（奪い合いでなく各自取得。両者が同時に 1.5 をキープし得る）。星は何度でも取り合える権利ポイント。

### ルール③ 0.5石が動ける（ムーブ）

0.5 石だけが動ける（1 石は動かせない）。移動は通常の 1 手。移動範囲は隣接 8 マス。動かしたターンは置けない。移動先の結果も足し算。盤上の 0.5 石すべてが常に脅威になる。

**同時ムーブの特殊ケース**: 両者が同じ場所へ → トリス（3箇所同時に空く）／互いの石へ → スワップ（入れ替わり）。

### 連続配置禁止（コウ相当）

直前のターンに置いた・動かした場所には次のターン置けない（2 ターン後に解放）。相殺ループ防止。

## 現象の命名

ポア / ソリディファイ / **キャンセル（0.5 同士の相殺）** / ムーブ / スワップ / **プール（囲みが完成し液体が溜まる）** / **リーク（囲みが破れ決壊）** / チャージ / トリス。略称は自然発生に任せ、公式は正式名で統一。

> 旧「オーバーポア」（色奪い）と「ブレイク」（0.5 消滅）は廃止。色奪いは起こらず、0.5 同士の相殺は **Cancel（キャンセル）** に一本化した。

## デジタルならではの解決

- 物理碁のコミ（五目半）は「石が整数しか存在できない」制約への補正。リキッ碁は 0.5 石が自然に存在 → コミ不要。
- 地の量を液体量としてリアルタイムに可視化。
- **スコアは m³ 単位**（1 マス = 1m³）。「何目」ではなく「何 m³」。**水量＝地＝スコア**なので、盤上の水の総体積がそのまま得点になる。

## 技術方針

- Web アプリ。Three.js で描画。
- **石は柵（境界線・壁）として描く**（丸い碁石は使わない）。同色隣接＝実線、斜め＝隙間ある斜め線、1石＝硬い実線の柵、0.5石＝半透明で揺らぐ壊れやすい柵。
- **水は「一色の柵で囲われた領域」に溜まる地**として描く。囲いの安定度で標高が決まり（確定＝海抜0の凪の池、不安定＝高く今にも流れ出す、囲いの無い所＝乾く）、**流れ出る液体**としてシミュレートする。石を置くたび柵と水がリアルタイムに変形。液体はシェーダーで表現。
- スコア（m³）＝盤上の水の総体積。
