# リキッ碁 / Liqui-go — ゲームデザイン

## コンセプト

「物理の碁石では表現できなかった 0.5 石を、液体で実現した囲碁」。

サッカーを見て「あとから動かせる囲碁」と感じた観察が起点。スペースやエリアへの「効き」を可視化したい、という動機から発展した。

## 世界観

| 要素       | 物質                                   |
| ---------- | -------------------------------------- |
| 1 石       | 石製の枠。完全に固まった構造物         |
| 0.5 石     | まだ固まっていないセメント。柔らかく不安定 |
| 内側の液体 | 白・黒の流体。枠に閉じ込められて溜まる  |

液体の物理比喩で囲碁の状態を表す（確定した地＝海抜0の池、勢力が強い場所＝表面張力で盛り上がった液体、死に石＝水没、など）。

## 石の計算ルール — すべて足し算

黒をプラス、白をマイナスとして加算するだけ。セル値は `{-1, -0.5, 0, 0.5, 1}` の5値。

**石が触れ合う経路は2つだけ。** 着手は空き点にしか置けない。既にある石へ直接「置きにいく」ことは無い。

- **① 同時着手** — 両者が同じ空き点を同じターンに奪い合う（ルール①）。デルタが足し合わさる。
- **② ムーブ** — 0.5 が隣へ乗り込む（ルール③）。**1石は動けない**ので、1石のほうから相手の石へ触れにいくことはできない。

この2経路で起こる加算の全パターン:

| ぶつかり方            | 計算            | 結果              | 経路                  | 現象                    |
| --------------------- | --------------- | ----------------- | --------------------- | ----------------------- |
| 空きに1石             | `0 + 1 = 1`     | 1石               | 着手                  | place                   |
| 空きにポア            | `0 + 0.5 = 0.5` | 0.5石             | 着手                  | ポア (pour)             |
| 自分の0.5 + 自分の0.5 | `0.5 + 0.5 = 1` | 1石に確定         | ムーブ                | ソリディファイ          |
| 0.5 + 相手の0.5       | `0.5 - 0.5 = 0` | 両方消滅          | 同時着手 / ムーブ     | **キャンセル (cancel)** |
| 1石 + 相手の1石       | `1 - 1 = 0`     | 両方消滅          | 同時着手              | capture                 |
| 自分の1石 + 相手の0.5 | `1 - 0.5 = 0.5` | 自分の0.5へ削れる | 同時着手 / ムーブ迎撃 | reduce                  |

**1石が0.5とぶつかると必ず0.5に削れる**（1のまま残らない）。相手の0.5の上に1石を置いて色を奪う手は、1が動けず同時着手も空き点限定なので**起こらない**（旧「オーバーポア」は廃止。0.5 同士の相殺は **Cancel** に統一し、旧「ブレイク」も廃止）。

## 3つのルールセット（絡み合って成立する）

液体表示・境界線・3D 地形は**表示の改善**であってルール変更ではない。

### ルール① 同時プロット制

先手・後手なし。両者が次の手を同時に公開する。コミ不要（先手有利が構造的に存在しない）。衝突は全て足し算で処理。

### ルール② 1.5手（星パックマン）

星・天元に置いた瞬間、1.5 手の権利を獲得（ストック、上限 1.5）。星の石が取られても権利は消えない。相手が同じ星に置けば**相手も**（各自の権利を）1.5 に据える（奪い合いでなく各自取得。両者が同時に 1.5 をキープし得る）。星は何度でも取り合える権利ポイント。

### ルール③ 0.5石が動ける（ムーブ）

0.5 石だけが動ける（1 石は動かせない）。移動は通常の 1 手。移動範囲は隣接 8 マス。動かしたターンは置けない。移動先の結果も足し算。盤上の 0.5 石すべてが常に脅威になる。

**同時ムーブの特殊ケース**: 両者が同じ場所へ → トリス（3箇所同時に空く）／互いの石へ → スワップ（入れ替わり）。

### 連続配置禁止（コウ相当）

直前のターンに置いた・動かした場所には次のターン置けない（2 ターン後に解放）。相殺ループ防止。

## 現象の命名

ポア / ソリディファイ / **キャンセル（0.5 同士の相殺）** / ムーブ / スワップ / プール / リーク / チャージ / トリス。略称は自然発生に任せ、公式は正式名で統一。

> 旧「オーバーポア」（色奪い）と「ブレイク」（0.5 消滅）は廃止。色奪いは起こらず、0.5 同士の相殺は **Cancel（キャンセル）** に一本化した。

## デジタルならではの解決

- 物理碁のコミ（五目半）は「石が整数しか存在できない」制約への補正。リキッ碁は 0.5 石が自然に存在 → コミ不要。
- 地の量を液体量としてリアルタイムに可視化。
- **スコアは m³ 単位**（1 マス = 1m³）。「何目」ではなく「何 m³」。

## 技術方針

- Web アプリ。Three.js で 3D 地形レンダリング。
- 各交点の「確定度」をスコアリングして heightmap に変換。
- 石を置くたび地形がリアルタイムに変形。液体シミュレーションはシェーダーで表現。
